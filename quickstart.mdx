This page walks you through the minimal quickstart example from the Cursor Cookbook. In a few steps you will have a local Cursor agent running in Node.js, sending a prompt, and printing the streamed response to your terminal. The full example lives in `sdk/quickstart` in the cookbook repository.

<Steps>
  <Step title="Get a Cursor API key">
    Go to [cursor.com/dashboard/integrations](https://cursor.com/dashboard/integrations) and create an API key. You will set it as an environment variable in a later step.
  </Step>
  <Step title="Navigate to the quickstart directory">
    Clone the cookbook repository and navigate to the quickstart package.

    ```bash
    git clone https://github.com/cursor/cookbook.git
    cd cookbook/sdk/quickstart
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    pnpm install
    ```
  </Step>
  <Step title="Set your API key">
    ```bash
    export CURSOR_API_KEY="crsr_..."
    ```
  </Step>
  <Step title="Run the quickstart">
    ```bash
    pnpm dev
    ```

    The agent will analyse the current working directory and stream a one-paragraph description of the project to stdout.
  </Step>
</Steps>

## The full source

```typescript src/index.ts
import { Agent } from "@cursor/sdk"

const agent = await Agent.create({
  apiKey: process.env.CURSOR_API_KEY,
  name: "SDK quickstart",
  model: { id: process.env.CURSOR_MODEL ?? "composer-2" },
  local: { cwd: process.cwd() },
})

const prompt = "Explain this project in one paragraph."
const run = await agent.send(prompt)

for await (const event of run.stream()) {
  if (event.type !== "assistant") continue

  for (const block of event.message.content) {
    if (block.type === "text") {
      process.stdout.write(block.text)
    }
  }
}

await run.wait()
```

## What the code does

**`Agent.create()`** creates a new local agent. The `local: { cwd: process.cwd() }` option tells the agent to operate against the current working directory on your machine — the same directory you run the script from. The `model` field defaults to `composer-2` but can be overridden by setting `CURSOR_MODEL` in your environment.

**`agent.send(prompt)`** sends the prompt to the agent and returns a `Run` object. The run represents a single exchange in the agent's conversation history and is the handle through which you stream events and wait for completion.

**`run.stream()`** returns an async iterable of events emitted as the agent works. Events have a `type` field that identifies what kind of activity produced them. This example filters for `"assistant"` events, which carry the model's response.

**`event.message.content`** is an array of content blocks. Each block has a `type` field; this example handles `"text"` blocks by writing them to stdout as they arrive, producing incremental output instead of waiting for the full response.

**`run.wait()`** waits for the run to finish before the process exits.

## Understanding the output

`run.stream()` yields events throughout the life of a run, not just the final response. By checking `event.type !== "assistant"` and using `continue`, the loop silently skips tool calls, status updates, and other internal events and only acts on assistant messages. Within each assistant message, `event.message.content` may hold multiple blocks — a single response can interleave text with other block types — so the inner loop iterates over all blocks and writes only the ones where `block.type === "text"`.

<Tip>
  You can remove the `if (event.type !== "assistant") continue` guard and log all events to see everything the agent emits during a run.
</Tip>
