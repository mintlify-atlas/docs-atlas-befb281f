The Cursor SDK is a TypeScript API for running Cursor's coding agent from your own applications, scripts, and automation workflows. It supports the same agent across local workspaces and cloud runtimes, streams agent events as runs progress, and lets you manage prompts, models, conversation state, and run lifecycle from code. This cookbook collects small, focused examples that show the SDK in action — each one runnable in a few commands.

## Prerequisites

- Node.js 22 or newer
- A Cursor API key from [cursor.com/dashboard/integrations](https://cursor.com/dashboard/integrations)
- pnpm (used in all examples)

## How each example is structured

Every example in the `sdk/` directory is a self-contained package with its own `package.json` and `tsconfig.json`. Each one declares `@cursor/sdk` as a dependency and uses `tsx` to run TypeScript directly without a separate compile step. To run any example, install its dependencies with `pnpm install` and start it with `pnpm dev`.

<Note>
  Set the `CURSOR_API_KEY` environment variable before running any example. You can get a key from the [Cursor integrations dashboard](https://cursor.com/dashboard/integrations).
</Note>
